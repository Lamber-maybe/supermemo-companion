# Learner profile

Copy this file to `LEARNER.md` and edit it — either in the current working
directory (if you keep a dedicated study/notes folder you always work
from) or in your home directory as `~/LEARNER.md` (reachable regardless of
which project an agent is launched from). Agents read it before judging
material or coaching you. Keep it under ~20 lines — it is context, not a
diary.

- **Language:** e.g. "answer me in English; card fronts in English", 或
  “用中文和我交流；卡片正面用英文术语，背面附中文释义”
- **Core domains** (learn deeply, details included):
  e.g. network security, Linux internals
- **Adjacent domains** (map-level awareness only):
  e.g. cryptography theory, ML fundamentals
- **Peripheral** (offload by default; only worth cards on my explicit
  request): e.g. frontend frameworks, cloud billing
- **Current goals:** e.g. OSCP in 6 months; read papers without a dictionary
- **Daily review budget:** e.g. ~30 min, max ~15 new items/day
- **SRS tool & version:** e.g. SuperMemo 19 · or: Anki 24.x — decides
  card output formats and whether tool coaching applies
- **Card preferences:** e.g. prefer cloze style; anchors not precise values;
  no multiple-choice; cards delivered as import files (or: suggest
  wordings only, I type them myself)
- **Docs mirror (optional, SuperMemo):** local path to your own crawl of
  help.supermemo.org, e.g. `~/Documents/supermemo-doc/wiki`. A
  revision-pinned Main + Glossary mirror already ships in `docs/wiki/`.
  Set this only for a demonstrably fresher or broader crawl; ideally it
  should expose a manifest/snapshot so the agent can compare revisions
  rather than assuming the whole directory is newer.
