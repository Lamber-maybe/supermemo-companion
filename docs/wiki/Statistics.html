Introduction

[edit | edit source]

The Statistics window allows you to inspect the main learning statistics in the currently opened collection. Statistics can be viewed by pressing F5 (Window : Layout : Warrior layout).

Figure: The Statistics window (Toolkit : Statistics : Statistics) where the main learning process statistics of the currently opened collection can be readily inspected.

Caption

[edit | edit source]

The caption displays the name of the collection in square brackets. The presented collection is named [ALL].

Statistics context menu

[edit | edit source]

To open the context menu:

  • Right-click anywhere in the window
  • Click the first button in the toolbar

Context menu items:

  • Workload - the daily and the monthly calendar of repetitions
  • Report - save the contents of the Statistics window to a text file
  • Restore layout (Ctrl+F5) - restore the default windows layout in SuperMemo
  • Element window (Esc) - shift focus to the element window

Date

[edit | edit source]

The current date and day of the week. If this value is preceded with Night, it means that the new calendar day has already started but the old repetition day will not start until the time defined in Toolkit : Options : Learning : Midnight shift. When the midnight shift is passed, this field will display a red warning Time to close: Alt+F4. If you see that message, close/restart your collection to prevent collecting learning data with undefined repetition timing. In the example above, the picture snapshot was taken after midnight on Oct 22, 2023 (Sun)

First day

[edit | edit source]

The date on which the learning process started (i.e. the day on which the first element was memorized). The exemplary collection presented in the picture has been in use since December 15, 1987 (i.e. two days after the birth date of SuperMemo for DOS)

Period

[edit | edit source]

The number of days in the learning process (i.e. number of days between Date and First day).

Period=Date-First day

The presented collection has been in use for 35 years, 10 months and 7 days

Memorized

[edit | edit source]

The total number of elements introduced into the learning process with options such as Learn or Remember. If an item takes part in repetitions it is a memorized item. It does not mean it is a remembered item. A proportion of memorized items is always forgotten. The presented collection has 714,918 elements in the learning process and these elements make up 100.0% of all elements destined to enter the learning process, i.e. Memorized/(Memorized+Pending)=100.0. This indicates that Pending is 0 or not much more (Pending=3 here)(see below).

See video guide.

Memorized items

[edit | edit source]

The number of memorized items in the collection, and the proportion of memorized items among memorized elements. In the example above, 214,918 items take part in repetitions. These items make 30.1% of all elements taking part in the learning process (the remaining 69.9% of elements are memorized topics, memorized concepts or memorized tasks). The Retention field indicates that 91.861% of these items should be remembered at any given time.

See video guide.

Memorized topics

[edit | edit source]

The number of memorized topics, concepts and tasks, and their cumulative proportion among all memorized elements. In a well-balanced incremental reading process, topics should make a minority of elements served for review. If the proportion of topics increases, the retention drops, and the learning process may gradually start to resemble traditional learning where ineffective passive review predominates. You can store as many topics in your collection as you wish as long as you make sure that you limit their review by setting appropriate repetition sorting criteria (Learn : Sorting : Sorting criteria). In the picture, 500,000 topics make 69.9% of the material taking part in the learning process.

See video guide.

Memorized/Day

[edit | edit source]

The number of items memorized per day: (Memorized items)/Day. In the example, the average of 16.411 items have been memorized daily in the presented collection over the previous 36 years. This is typical of an average student assuming daily use of SuperMemo.

See video guide.

Total

[edit | edit source]

The number of items, topics, concepts and tasks in the collection. Two relationships hold true:

Total=Memorized+Pending+Dismissed

Total=Topics+Items (concepts and tasks are included in the Topics statistic)

Deleted elements do not contribute to the total count of elements in the collection. In the picture, the presented collection is made of 817,817 elements (largest collections reported by users reached well above a million elements)

See video guide

Items

[edit | edit source]

The number of items in the collection. In the example, the collection includes 219,022 items. These include dismissed items that might be used as reference, or as record of past learning, or just as a souvenir

Topics

[edit | edit source]

The number of topics, concepts and tasks in the collection. The presented collection includes 598,795 topics, concepts and tasks

Outstanding

[edit | edit source]

The number of outstanding items, outstanding topics and final drill items scheduled for repetition on this given day. The first number (before the plus sign) indicates the number of items scheduled for this given day and not yet processed. The second number (after +) indicates the number of topics scheduled for review for this day. The third number (after the second +), if present, indicates the number of items that have already been repeated today but scored less than Good (4). Those are the items that make up the final drill queue. The final drill queue is built only if Skip final drill is unchecked in Toolkit : Options : Learning. In the presented collection, there are still 30 items scheduled for repetition on Oct 22, 2023. There are also 19 topics scheduled for review on that day as part of the incremental reading process. There are no elements in the final drill queue (the third component of the Outstanding parameter is missing)

Review

[edit | edit source]

The number of elements scheduled for subset review (e.g. elements in the neural review in Learn : Go neural, elements in branch repetitions in Contents : Learn, elements in browser subset repetitions in the browser's Learn, elements in the random test queue in Toolkit : Random test, etc.). The display may have a form of Neuro=<elements to do> in neural review, or <items to do>+<topics to do>+<pending to do>+(<subset description>) in subset review, or <elements unprocessed>/<all elements in the test> in random tests. Here 244 items remain in the neural queue used in neural creativity

Protection

[edit | edit source]

The current day's degree of processing of the top priority material. Important: As the statistic is taken from the top of the queue of outstanding items or topics (not the top of outstanding queue, which is a randomized mix of the two), if you change the priority of the top item, you will see a false value in Statistics until you review that item of changed priority (this behavior is by design to prevent the need to scan the entire queue at each update to statistics). In the example, only the first 58 top priority items, and the first 48 top priority topics have been processed. 58 item protection does not mean going through 58 items at the start of the outstanding queue (usually many more). It means that the highest priority of unprocessed items in the queue corresponds with the position 58 in the queue.

See Priority Protection: The Ultimate Litmus Test See Priority Protection Stat for SM18 & SM19 See Why You Have A Low Priority Protection See Sudden, Large Drop in Priority Protection See Chronically Low Priority Protection

Retention

[edit | edit source]

The estimated average knowledge retention in the collection. Retention for high-priority items should be higher than the one listed. Retention for low-priority items may be much lower, driving the average down. To inspect the retention of top-priority material, see Toolkit : Statistics : Analysis : Graphs : Forgetting index vs. Priority. In the example, 91.861% of the material should be recalled in a random test on all elements in the collection at any time. You can test your retention using random tests and see if SuperMemo's estimates are accurate. This statistic may be overly optimistic if you have recently abused rescheduling tools such as Postpone or Mercy.

See Understanding the Retention Rate in Spaced Repetition

Measured FI

[edit | edit source]

The value of the measured forgetting index as recorded during repetitions. The number in the parentheses indicates Measured FI for the day. In collections with heavy item overload, measured forgetting index may be much lower than the overall measured forgetting index for the entire collection due to the fact that repetitions include primarily high-priority material that is not subject to postpones. It can also be lower than the requested forgetting index when transitioning from more randomized to more prioritized sorting (as determined by the sorting criteria) or when knowledge formulation and mnemonic skills improve before this fact can be reflected in the forgetting curve. It is also not uncommon to have Measured FI higher than Average FI. This is due to three factors:

  1. every user will experience delays in repetitions (e.g. as a result of using Postpone),
  2. low-priority material in the overloaded incremental reading process is scheduled in intervals longer than optimum intervals,
  3. SuperMemo imposes some constraints on the length of intervals that result in repetitions later than determined by the forgetting index. The constraints in computing intervals, for example, prevent the new interval from being shorter than the old one (assuming the item has not been forgotten). For low values of the forgetting index and for difficult items, the new optimum interval might often be shorter than the old one! Measured FI can be reset with Toolkit : Statistics : Reset parameters : Forgetting index record. Resetting will resume measurements from scratch

In the presented example, an average of 15.82% of item repetitions end with a grade less than Pass (3) (since the measured forgetting index record has last been reset). On Oct 22, 2023, no repetition ended in failure yet (i.e. with a grade less than Pass (3))

R-Metric

[edit | edit source]

The absolute measure of performance of two spaced repetition algorithms based on their ability to predict recall before a grade is scored. In SuperMemo 20, R-Metric is used to compare Algorithm SM-19 against the weighted pack of algorithms competing in Algorithm Arena (comprising Algorithm SM-2, Algorithm SM-15, Algorithm SM-19, Algorithm SM-20, and FSRS). It is shown as percentage in Statistics and Toolkit : Statistics : Analysis : Use : Efficiency : R-Metric. R-Metric is a difference between the performance of the two sides: R-Metric=LSRM(Alg-19)-LSRM(Arena), where LSRM is the least squares predicted recall measure for a given algorithm. R-Metric greater than zero shows superiority of Algorithm Arena over Algorithm SM-19. R-Metric less than zero indicates underperformance of the Arena pack relative to Algorithm SM-19. LSRM is a square root of the average of squared absolute differences in recall predictions: abs(Recall-PredictedRecall), where Recall is 0 for failing grades and Recall is 1 for passing grades. PredictedRecall is a prediction issued by the algorithm before the repetition. In Algorithm SM-19, the prediction is a weighted average of the value taken from the Recall[] matrix, and R (retrievability) computed from S (stability) and the used interval. The weight used is based on prior repetition cases which inform of the significance of the Recall[] matrix prediction (the prediction becomes more meaningful with more prior repetition data).

  • R-Metric=LSRM(Alg-19)-LSRM(Arena)
  • LSRM(Alg)=sqrt(sum(sqr(ARD)))
  • ARD=abs(Recall-PredictedRecall)
  • Recall is 1 for Grade>=3, and 0 for Grade<3
  • PredictedRecall in Algorithm SM-19 is:
    • weight*Recall[]+(1-weight)*R
      • weight (0..1) depends on the number of prior repetition cases
      • R=exp(-kt/s)

In this case, R-Metric of 36.8691% shows a huge advantage of the Algorithm Arena weighted pack over Algorithm SM-19. Both negative and positive values are good news for you. Spaced repetition will still do marvels to your memory (picture is out of date; R-metric rarely goes above 5%; the contest is very close)

Alarm

[edit | edit source]

The time left till the next alarm, and the hour at which the alarm will ring (to learn more about alarms see: Plan). This field is editable. To change the alarm setting, click the field and type in the new time in minutes (e.g. 21.5 will set the alarm to sound in 21 minutes and 30 seconds). To end editing, press Enter. In the example, the alarm will sound off in 31 minutes and 11 seconds at 06:49:15

Burden

[edit | edit source]

The estimate of the average number of items and topics that have to be repeated per day. This value is equal to the sum of all interval reciprocals (i.e. 1/interval). The interpretation of this number is that every item with interval of 100 days is on average repeated 1/100 times per day. Thus the sum of interval reciprocals is a good indicator of the total repetitions workload in the collection. The presented collection requires 155 item repetitions per day and 372 topic reviews per day. In incremental reading, it is not unusual to have many more elements in the process than one can handle. Auto-postpone can be used to unload the excess of topics as well as to reduce the load of low-priority items. Postpone skews the Burden statistic. Topics often crowd at lower intervals and are regularly reshuffled with Postpone or Auto-postpone

Burden +/-

[edit | edit source]

The change of the Burden parameter above on a given day. Here, on Oct 22, 2023, the average number of expected daily repetitions was increased by 4.322 items due to forgetting or due to adding new items. The topic load was decreased by 7.707 topics). Exemplary interpretation of a burden change: Let's say Burden dropped by 39 (burden change of -39). To reduce the burden by 39, one would need to review 78 elements with an interval increase from 1 to 2 days (78*0.5=39). However, one could equally well execute Postpone on 2344 elements with interval increase from 10 to 12 days (2344*(1/10-1/12)=39)

Avg Workload

[edit | edit source]

The average time spent on responding to questions per day (from the first day of learning). For the presented collection, the student spent an average of 14 minutes and 58 seconds per day on answering items over the period spanning 35 years 10 months and 7 days between Dec 15, 1987 and Oct 22, 2023

Exp Workload

[edit | edit source]

The estimation of the average daily time used for responding to questions in a given collection.

Exp Workload=(Item Burden)*Avg time

In the presented collection, 155 item repetitions per day (Burden) taking 10.536 seconds each (Avg time) result in a daily repetition time estimated at 27 minutes and 13 seconds. A real learning time may be twice longer due to grading, editing, reviewing the collection and various interruptions. In incremental reading, the learning time will increase further due to topic review that is not taken into account in the Exp Workload parameter. The real learning time may also be cut if Postpone is often used. The user is not obliged to make all repetitions

[edit | edit source]

Here the total time needed to respond to questions on Oct 22, 2023 was 2 minutes and 10 seconds. On the same day, the presented collection has been running for 55 minutes and 22 seconds (this value will increase even if you simply keep SuperMemo running while not learning with that particular collection). Avg time

[edit | edit source]

The average response time in seconds. This is the time that elapses between displaying the question (or equivalent) and choosing Show answer (or equivalent). The timer does not stop if you start editing the question before pressing Show answer. The averaged answer time is limited to 30 second just in case you left the keyboard for coffee. In the presented collection, the average time to answer a single question is around 10.536 seconds. If this number grows beyond 15-20 seconds, you may need to analyze your learning material if it is not overly difficult or badly structured. See: details

Total time

[edit | edit source]

The total time taken by responding to questions in the collection. This time cannot be accurately measured for collections created with SuperMemo 98 or earlier (the measurements were made possible only in SuperMemo 99). If you upgrade older collections, this number will roughly be guessed for you. SuperMemo will derive this time from the total number of items, average number of repetitions, average number of lapses, and the average repetition time. In the presented example, answering questions during repetitions took the total of over 136 days and 3 hours in nearly 36 years of learning

Lapses

[edit | edit source]

The average number of times individual items have been forgotten in the collection (only memorized items are averaged). The number in parentheses shows the number of lapses on a given day. Here an average item has been forgotten 0.48249 times. On Oct 22, 2023, no item has thus far been graded less than Pass (3)

Speed

[edit | edit source]

The average knowledge acquisition rate, which is defined as the number of items memorized per year per minute of daily work. Only the time used for answering item questions is counted. Initially, this value may be as high as 100,000 items/year/minute (esp. if you enthusiastically start working with the program before truly measuring its limitations, and the limitations of human memory). This parameter should later stabilize between 40 and 400 items/year/minute. It may be overly optimistic in collection with a heavy overload.

Speed=(Memorized items/Day)/(Repetitions time)*365

In the presented collection, every minute of work per day resulted in 367.8 new items memorized each year (i.e. roughly one item per day per minute).

See Understanding the Speed stat in SuperMemo


Avg Cost

[edit | edit source]

The cost in time of memorizing a single item, i.e. total learning time divided by the number of memorized items.

Avg Cost=Total time/Memorized items

In the presented example, the total repetition time per single item is 59 seconds, which is the amount of time it has contributed to the total of over 136 days of continual repetitions. The cost of editing, collection restructuring, incremental reading, etc. is not included in the Avg Cost parameter

Exp Cost

[edit | edit source]

The daily repetition time per each newly memorized item assuming no postpones.

Exp Cost=Exp Workload/(Memorized items/Day)

In the presented collection, each of the 16 newly memorized items per day contributes 1 minute and 39 seconds of repetitions to the total workload of over 27 minutes per day. As this value is derived from Burden, it may be highly overestimated if you use Postpone a lot (e.g. in incremental reading)

Pending

[edit | edit source]

The number of elements (topics or items) that have not yet been introduced into the learning process and await memorization (with operations such as Learn, Remember, Schedule, etc). All pending elements are kept in the pending queue that determines the sequence of learning new elements. Dismissed elements are not kept in the pending queue. In the example, the collection contains 3 pending elements. With incremental reading, the role of the pending queue in SuperMemo is diminishing.

See video guide-

Dismissed

[edit | edit source]

The number of elements (items, topics, concepts or tasks) that have been excluded from the learning process and are kept only as reference material, folders in the knowledge tree, or tasklist elements. Dismissed items are neither pending nor memorized. In the example, 102,896 elements have been dismissed.

See video guide.

Average FI

[edit | edit source]

The average requested forgetting index in the entire collection (the number in parentheses is the default forgetting index). If the forgetting index of individual elements is not changed manually, Average FI is equal to the default forgetting index as set in Toolkit : Options : Learning : Forgetting index (default). The default forgetting index is the requested forgetting index given to all new items added to the collection. Forgetting index, in general, is the proportion of items that are not remembered during repetitions. The lower the value of the forgetting index the better the recall of the element, but the more repetitions will be needed to keep it in memory. Optimum value of the forgetting index falls into the range from 7% to 13%. Too low a forgetting index makes learning too tiresome due to a prohibitively large number of repetitions. All elements can have their desired forgetting index set individually. The easiest way to change the forgetting index of a large number of elements is to use Forgetting index option among subset operations. In the presented example, the average forgetting index is 10.00% while the default forgetting index is 10%. See: Using forgetting index

Completion

[edit | edit source]

The expected date on which all elements from the pending queue will be memorized assuming the present rate of learning new items. This parameter is particularly useful if you are memorizing large ready-made collections such as Advanced English. For Pending=3, the value of this field is today.

Completion=Date+(Pending/(Memorized items/Day))

A-Factor

[edit | edit source]

The average value of A-Factor among memorized items (I) and topics (T) in the collection. For items, A-Factor is a measure of difficulty in Algorithm SM-15. The higher the A-Factor, the easier the item. For topics, A-Factor is the number by which the current interval should be multiplied to get the value of the next interval. In the presented collection, the average A-Factor for items is 4.09. This indicates that the collection is rather well-structured and the material is thus relatively easy to remember. The average A-Factor for topics is 1.219

Additional notes

[edit | edit source]
  • Items are added to the final drill not only during standard repetitions when you grade an element below Good (4). Operations such as Remember (Ctrl+M), Cloze (Alt+Z), and Add to drill (Shift+Ctrl+D) will also add to the final drill queue. The final drill queue is created automatically only if you uncheck Toolkit : Options : Learning : Skip final drill
  • Some fields of the Statistics window can be edited. For example: Alarm, Total time, Rep count, etc. To edit an entry, click it, type the new value and press Enter. If the entry cannot be modified SuperMemo will warn you (e.g. "Retention entry cannot be modified").
  • See Survey 1994 and Survey 1999 for some interesting notes about the speed of learning reached with SuperMemo
