Registries in SuperMemo are sorted sets of objects used such as texts, images, sounds, fonts, etc. By using registries, SuperMemo reduces the overall size of collections, and makes it easy to reuse objects already introduced into the current collection. For more information about registries see: Using registries.

All existing registries can be accessed from the Search menu.

A registry window is used to access individual members of a registry.

Parts of the registry window

Image names are displayed on the left. Most of pictures are stored in the JP(E)G format. Other formats show in different colors: GIFs in orange, PNGs in yellow and BMPs in lime. The list of elements using a given picture can be displayed in the bottom-right panel by clicking List at the very bottom of the registry window. The status bar displays selected information about the current image. It is the 3637th image in the alphabetical order. It is reused by 55 elements. It is located on physical position 2270 in the registry. It occupies the file slot 196353 in the [ELEMENTS] subfolder. Its path and filename in the [ELEMENTS] subfolder is displayed next. The file is 121,712 bytes large. It was imported or created on Oct 21, 2006 at 17:54:34.

  • caption - name of the registry and the total number of registry members. Here the image registry keeps 21,645 pictures
  • registry toolbar (below the caption) - tool buttons providing mouse shortcuts to the most often used registry options (incl. access to the registry menu)
  • editing field (under the toolbar) - field used in incremental search, i.e. searching for members by typing in initial characters of their names
  • list of member names (left panel) - alphabetically sorted list of names of individual registry members. Here the list displays images from "Citric acid cycle (Krebs cycle, TCA cycle)" to "Climate scientists poll". The scrollbar on the right side of the list box shows the approximate position of this subset of images in the alphabetical list
  • object display field (right top panel) - field that is used to display the object that is currently selected in the list of members (on the left side). In some registries this field is empty (e.g. sound registry, program registry, etc.) or is divided into two parts (e.g. text registry displays text translations, lexical registry displays phonetic transcription of individual words, translation registry displays the original texts, etc.). Here the Citric acid cycle (Krebs cycle, TCA cycle) is presented
  • element list (right bottom panel) - upon clicking List at the bottom of the registry window, the element list is filled out with all elements that use a given registry member.
  • status bar (above the button panel) - information about the currently selected member
  • button panel (at the very bottom)

A right click over the registry window opens the registry menu with advanced operations on the registry.

Status bar

[edit | edit source]
  • registry index (i.e. the ordinal position of the registry member in the registry)
  • number of elements that use the member
  • physical position in the registry
  • number of the filespace slot (if any) - a numbered position of a file in the [ELEMENTS] subfolder. The first file will get the slot one, second slot two, etc. (e.g. 1.jpg, 2.htm, 3.wav, etc.), and
  • (if relevant) path to the registry member file with the file size In the example above, the picture is alphabetically the 3637th picture in the registry, it is used by 55 elements, it is physically placed at the position 2270 in the registry, and it uses the filespace slot number 196,353. The actual jpg file is located in the folder c:\colls\All\elements\21\24\15\ and is named 196353.jpg (note that the name is the same as the number of the filespace slot). The size of this file is 121,712 bytes and it was created on Oct 21, 2006 at 17:54:34

Button panel

[edit | edit source]
  • Browse - display all elements that use the currently selected registry member in an element browser. Note that the Browse button on the toolbar can be used to display all elements that use all registry members belonging to the current subset
  • List - list the elements that use the currently selected registry member (the list is displayed in the element list at the bottom right panel of the window)
  • Go to - go to the first element using the currently selected registry member
  • Neural (Alt+N) - execute the neural review over the selected registry member
  • Rename (Alt+R) - change the name of the currently selected registry member
  • Insert - create a new component with a link to the currently selected member. SuperMemo will ask you if the component should be created in the current element displayed in the element window or if a new element should be created
  • Add - add a new registry member to the registry
  • Accept - link the currently selected registry member with the requesting source (e.g. if you choose Links : Registry member on the sound component menu, the component will be linked with a sound object associated with the sound registry member selected at the moment of pressing Accept)

SuperMemo 中的注册表是所使用的对象的排序集，例如文本、图像、声音、字体等。通过使用注册表，SuperMemo 减少了总体大小收藏品，并且可以轻松重用已引入当前集合的对象。有关注册表的更多信息，请参阅：使用注册表。

所有现有的注册表都可以从搜索菜单。

注册表窗口用于访问个人登记处的成员。

注册表窗口的部分内容

图：一张图片注册表是一组按名称排序的图像文件。该图展示了一个典型的图像注册表。共有 21,645 张图片注册表（如标题所示）。图像名称显示在左侧。 大部分图片都存储在JP(E)G格式。其他格式以不同颜色显示：动图在橙色,PNG在黄色和骨形态发生蛋白在石灰。带有勾号的图片可以提取成图片子集（例如，为了说明所选的元素）。当前显示的图片是柠檬酸循环（克雷布斯循环、TCA循环）之一。个人元素在 SuperMemo 中可以专注于这张大图片的一小部分。使用给定图片的元素列表可以通过单击显示在右下面板中列表在最底部注册表窗口。状态栏显示有关当前图像的选定信息。这是按字母顺序排列的第 3637 张图像。被 55 人重复使用元素。它位于物理位置2270注册表。它占用了文件槽196353[元素] 子文件夹。接下来显示其在 [ELEMENTS] 子文件夹中的路径和文件名。该文件大小为 121,712 字节。它是在 2006 年 10 月 21 日 17:54:34 导入或创建的。

  • 标题 - 名称注册表和总数登记册成员。这里的图像注册表保存了 21,645 张图片
  • 注册表工具栏（标题下方）- 工具按钮，提供最常用的注册表选项的鼠标快捷方式（包括访问注册表菜单）
  • 编辑字段（工具栏下方）- 用于增量搜索的字段，即搜索会员通过输入名字的首字母
  • 成员姓名列表（左面板） - 按字母顺序排序的个人姓名列表登记册成员。此处列表显示从“柠檬酸循环（克雷布斯循环、TCA 循环）”到“气候科学家民意调查”的图像。列表框右侧的滚动条显示该图像子集在字母列表中的大致位置
  • 对象显示字段（右上面板）- 用于显示当前在列表中选择的对象的字段会员（在左侧）。在一些登记处该字段为空（例如声音注册表、程序注册表等）或分为两部分（例如文本注册表显示文本翻译，词汇注册表显示单个单词的音标，翻译注册表显示原始文本等）。这里介绍了柠檬酸循环（克雷布斯循环、TCA 循环）
  • 元素列表（右下面板）- 单击后列表在注册表窗口的底部，元素列表填写了所有元素使用给定的注册会员。
  • 状态栏（按钮面板上方）- 有关当前所选内容的信息会员
  • 按钮面板（在最底部）

右键单击注册表窗口将打开注册表菜单与先进的操作注册表。

按键面板

[编辑|编辑源代码]
  • 浏览- 显示全部元素使用当前选择的注册会员在一个元素浏览器。请注意，浏览按钮于工具栏可以用来显示所有元素使用所有登记册成员属于当前的子集
  • 列表- 列出元素使用当前选择的注册会员（列表显示在窗口右下面板的元素列表中）
  • 前往- 转到第一个元素使用当前选择的注册会员
  • 神经（Alt+N) - 执行神经回顾超过所选的注册会员
  • 重命名（Alt+R) - 更改当前选定的名称注册会员
  • 插入- 创建一个新的组件带有指向当前所选内容的链接会员。 SuperMemo 会询问您是否应在当前创建组件元素显示在元素窗口或者是否应该创建一个新元素
  • 添加- 添加一个新的注册会员到注册表
  • 接受- 链接当前选择的注册会员与请求源（例如，如果您选择友情链接 : 注册会员在声音上组件菜单，该组件将与与按下时选择的声音注册表成员关联的声音对象链接接受）
