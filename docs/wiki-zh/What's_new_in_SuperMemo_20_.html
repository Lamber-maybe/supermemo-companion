New features

[edit | edit source]

Performance

[edit | edit source]
  • 64-bit: native Windows 11 APIs, hardened security, modern libraries, and an architecture ready for the next 20 years
  • High‑DPI: crisp rendering, correct scaling on 4K/8K monitors, and a UI that finally matches your hardware
  • WebView component
  • YouTube in WebView with original YouTube references
  • Recompiled with Delphi 13
  • Registry text file size limit increased to 4 GB (all registries, all *.rtx files)

Algorithm

[edit | edit source]
  • Algorithm Arena for the optimized balance between five algorithms: Algorithm SM-2, SM-15, SM-19, SM-20 and FSRS (see why?)
  • new spaced repetition algorithm: Algorithm SM-20
  • spaced repetition algorithm: FSRS by Jarrett Ye
  • R-Metric redefined to measure the power of algorithms in Algorithm Arena
  • Heatmap of weighed up optimization indications for five algorithms: SM-2, SM-15, SM-19, SM-20 and FSRS
  • Accounting for "bad" forgetting curves in Algorithm SM-19 and Algorithm SM-20
  • Algorithmic calibration curves (see: Analysis)
  • Active final drill (cramming for exams will not disrupt your learning process)
  • All recall prediction optimizations use universal metric as opposed to standard machine learning metrics (Brier, log-loss, etc.)
  • Use graphs in Analysis to track algorithmic metrics (incl. the Arena)

Imports

[edit | edit source]
  • PDF import in native format (for incremental reading)
  • Selection import mode (import only portions of articles from the web)
  • Full screen import mode (for intractable pages or pictures)
  • EPub imports
  • EML (Email) imports
  • Improved duplicate detection during import by automatically stripping advertising and tracking parameters from URLs (e.g. UTM, gclid, fbclid, affiliate IDs)
  • YouTube references and description retrieved via YouTube API
  • WebView makes it easy to browse the web and generate extract from pages on the web (see: video)

  • Explain extract for extracts with a short AI explanation

Improved Algorithm SM-19

[edit | edit source]

  • Computing a memory model for a subset

  • Capping MaxA in power regression in proportion to AverageRecall to limit first optimum interval to 365 days
  • Export option for forgetting curve analysis
  • Multiple metrics for comparing algorithms SM-2, SM-15, SM-19, SM-20 and FSRS

Bugs fixed

[edit | edit source]
  • Mathematical formulas would not import from Wikipedia (using WebView component resolves this problem)
  • Wrong formatting of retrievability in post-lapse interval estimates
  • Images pasted directly into HTML components are now properly tracked in the image registry. Previously, these images were left behind as orphaned files in the collection's filespace when the topic was deleted or marked as Done (see: bug report)
  • The first repetition date record in items would get lost when transferring elements between collections
  • Lazy loading images would not show in downloaded pages
  • <base href> would not be respected in resolving URLs
  • Images in <iframe> would not show in imported pages
  • The A shortcut now correctly triggers editing of the answer in Contents
  • Resolved errors occuring during the duplication of concepts

Annoyances fixed

[edit | edit source]
  • Interval choice dialog could not be cancelled (see also: How to get rid of the interval choice dialog?)
  • One key press could delete the entire activity entry in Plan. Now Enter must precede editing (not everyone is happy)
  • Yellow bar at the bottom of Statistics for certain themes
  • The long list of missing hints and misspelled hints collected by Alexis and Palace (thank you) was processed in one go with the help of AI

Warnings

[edit | edit source]
  • High-DPI support may make the 64-bit SuperMemo UI feel too large on lower resolution screens (4K is recommended, but 1440p or 1080p should be usable).
  • Some users may still prefer the 32-bit application (e.g. for smaller UI elements at lower resolutions). You can download a separate 32-bit version, though its updates may lag slightly behind the main 64-bit release (download [here] xxx link to be provided when ready)
  • SuperMemo 20 opens SuperMemo 19 collections, but SuperMemo 19 may struggle with SuperMemo 20 collection (e.g. due to the WebView component)
  • RTF components are now deprecated. You can still use old collections, but further use will be more difficult with some options removed
  • Some AI and YouTube options may require your own AppKey (available free from Google)
  • Optimizing FSRS or SM20 algorithms may require access to SuperMemo.com API

Interval heatmap

[edit | edit source]

Figure: Only very long intervals magnify significant differences between various spaced repetition algorithms, incl. those that score very high on the universal metric benchmarks (e.g. SM-19 and SM-20). In the presented example, a very easy item from Advanced English achieved a very long interval of over 26 years in just 5 repetitions. The colored heatmap shows a weighted optimum with the darkest red color showing the best consensus of 32 years and 3 months. At the extreme ends of disagreement, Algorithm SM-2 points to an optimum of 66 years because it is unable to sufficiently reduce its E-Factor in just 5 repetitions (for successive grades 4 it would still imply 2.5-fold increase in interval). Unoptimized FSRS attempts to reconstruct the memory status using the three component model of memory and the history of repetitions. Each repetition adds to the potential error; however, its conservative 9-year interval proposition is a pretty safe bet that most users will still find as excessively long. In this particular case, optimized Algorithm SM-20 provided the closest bet to the weighted optimum of 32 years, but the actual "winner" will differ from repetition to repetition. Both FSRS and Algorithm SM-19 propose shortening the next interval; however, SuperMemo 20 disallows this (due to anti-cramming measures). The heatmap is a form of asymmetric bell-shaped distribution, and algorithmic weighing will receive microscopic nudges along the learning process depending on the R-metric scores. The black area on the left represents the previously used interval of 26 years. Important! Users can always override the recommendations of SuperMemo and set a shorter interval, e.g., with Ctrl+Shift+R

Algorithmic metrics

[edit | edit source]

Figure: Universal metric is computed by cross-comparison of predicted recall in retrievability categories computed by a competing algorithm. The closer the algorithm adheres to the black diagonal, the better its performance in a particular collection. Unoptimized Algorithm SM-20 and FSRS may show substantially worse performances as compared with optimized SM-20 and FSRS

区间热图

[编辑|编辑源代码]

图：只有非常长的间隔才能放大不同类型之间的显着差异间隔重复算法，包括那些在通用度量基准（例如 SM-19 和 SM-20）。在所提供的示例中，一个非常简单的项目来自高级英语仅用了5年时间就实现了超过26年的超长间隔重复。彩色热图显示了加权最佳值，其中最深的红色显示了 32 年零 3 个月的最佳共识。在分歧的极端情况下，算法 SM-2指出最佳时间为 66 年，因为它无法充分减少其E因子只需 5 次重复（对于连续的 4 年级，这仍然意味着间隔增加 2.5 倍）。未优化FSRS尝试使用重建内存状态记忆三成分模型和重复的历史。每次重复都会增加潜在的错误；然而，其保守的 9 年间隔主张是一个相当安全的赌注，大多数用户仍然会发现它过长。在这种特殊情况下，优化算法 SM-20提供了最接近 32 年加权最佳值的赌注，但实际的“赢家”会因重复而异。两者都FSRS和算法 SM-19建议缩短下次间隔；然而，超级备忘录 20不允许这样做（由于反填充措施）。热图是一种不对称钟形分布的形式，算法权重将在学习过程中受到微观推动，具体取决于R-度量分数。左边的黑色区域代表之前使用的26年间隔。重要！用户始终可以覆盖 SuperMemo 的建议并设置更短的间隔，例如，Ctrl+Shift+R

图：通用度量是通过对竞争算法计算出的可检索性类别中的预测召回率进行交叉比较来计算的。算法越接近黑色对角线，其在特定集合中的性能就越好。未优化算法 SM-20和FSRS与优化的 SM-20 和 FSRS 相比，可能表现出明显更差的性能
