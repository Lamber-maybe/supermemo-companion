Accessing the registry menu

[edit | edit source]

All registries can be inspected via the Search submenu on the main menu.

Most important operations on registries are available from the registry menu. To open this menu:

  • right-click over the registry window,
  • press Alt+F10,
  • click the registry menu button on the registry toolbar, or
  • press Apps Key () (context menu key left of the right Ctrl key).

Registry menu items

[edit | edit source]

The registry menu includes the following options:

Go neural

[edit | edit source]
  • Go neural (Alt+N) - begin a neural review of a concept or associated registry members

View

[edit | edit source]
  • View
    • List elements - list all elements that use the currently selected registry member in the element list (bottom right of the registry window)
    • View element - display the element selected in the element list or display the first user of the currently selected registry member (if the element list is not shown). The element is shown in the element window without closing the registry. Note that elements are displayed each time you click on the entries in the element list (which can be shown with List elements above)
    • Go to element - same as View element but the element window is put to the front of the registry window
    • Reference - view the reference associated with the currently displayed image in the image registry
    • Browse current (Ctrl+B) - display all elements that use the current registry member in a browser
    • Browse selected (Shift+Ctrl+B) - display all elements that use the selected registry members, i.e. members with a checkmark on the left
    • Browse all - display all elements that use all the members in the current registry subset in the element browser
    • Random pick (F11) - jump to a randomly picked registry member in the current registry subset. This option can be used to quickly review the registry.
    • Go to position (Ctrl+G) - go to a registry member by providing its physical position in the registry (e.g. File : Repair collection will often report registry problems by referring to individual members by their physical position)
    • Path - display the path to the selected registry object file (e.g. image file). Show the availability of the file in the primary and secondary storage (if these two differ), linked file path (if any), and file sizes
    • Play (Ctrl+F10) - play the sound or video associated with the current registry member in sound or video registry
    • Encoding - display UTF-8 codes of the registry member name
    • Clear (Esc) - this option is useful in backtracking your operations on the registry window. This is the sequence of actions done by Clear:
      1. jump from the list of elements to the list of registry members (if the focus is on the list of elements)
      2. jump from the object editing fields to the list of registry members (if the focus is on one of the editing fields)
      3. jump from the member list to the editing combo-box at the top of the registry window (if the focus is on the member list)
      4. clear the contents of the editing combo-box at the top of the registry window if the focus is on the editing combo-box
      5. close the registry window if the focus is on the editing combo-box and the combo-box is empty

Edit

[edit | edit source]
  • Edit
    • Paste - paste an object from the clipboard into the registry
    • Paste over - replace the object the selected registry member points to with a new version available from the clipboard. For example, you can use this option to replace one picture with another. The replacement will show throughout the collection
    • Overwrite file - replace the current member's object file with another file. This option is similar to Paste over with a difference that while Paste over uses the data from the clipboard, if you use Overwrite file, you point the location of the new version of the file on your disk.
    • Edit text (Ctrl+E) - edit the text currently displayed in the text registry
    • Edit translation (Ctrl+T) - edit the translation associated with the current text registry member. Note that in translation registries, the meaning of Edit text and Edit translation is reversed, and Edit text is actually used to edit the translation
    • Copy path - copy the path to the file associated with the current registry member to the clipboard

Search

[edit | edit source]
  • Search
    • Find texts (Ctrl+S) - find all members in the current registry subset whose names contain a given string. Place the search results in a newly created registry subset. Use Ctrl+S to search all registry members for a string. Use it also for AND-search
    • Find first (Ctrl+F) - find the first registry member in the current subset whose name contains a given string. Search begins with the member that follows the currently selected member. You will find the shortcut for Find first, Ctrl+F, very useful here
    • Find next (F3) - repeat the last Find first operation (you can repeat search with F3)
    • Search and replace (Ctrl+R) - find members in the current registry subset whose names contain a given string and replace this string with another string. You can repeat this operation on all individual members with individual confirmations or replace all strings in one go. You can do global search and replace on texts in your collection by using the text registry. Caution: all registry member names must be unique, if your replace operation generates a name that already exists, the new name will be modified by SuperMemo to make it unique. Usually this will result in the appearance of some meaningless code at the end of your new name (e.g. Geography#, Geography@, etc.)
    • Apply filter - process the texts in the current registry with a filter file (e.g. replace a set of strings with another set of strings, remove delimited comments, etc.).

Rename

[edit | edit source]
  • Rename (Alt+R) - rename the currently selected registry member

Delete

[edit | edit source]
  • Delete (Del) - delete the currently selected registry member

Tools

[edit | edit source]
  • Tools
    • Import files - import a number of files into the registry. For example, if you have a set of ready-made pictures or sound files you can import them all to image or sound registry respectively before you use them in your elements
    • Import text - import the text with registry member names generated by Export as text (below). For example, after spell-checking
    • Export file - export the file associated with the current member
    • Export as text - export all registry member names into a text file. Most often you will want to do it to spell-check the contents of the registry with your favorite spellchecker
    • Sort for frequency - sort the registry for the number of elements that use individual members
    • Sort for size - sort the registry for the size of files associated with individual members (e.g. to delete largest pictures from your bloated collection)
    • Sort for priority - sort the registry for member priority. The priority here is determined by the ratio: (number of users)/(file size). This option is useful if you want to find the least used members that take most space (for example, Advanced English 97 sound files have been trimmed with this option to fit the 650MB CD-ROM size limit)
    • Externalize files - export all registry files to secondary storage
    • Internalize files - move all files associated with the registry from the secondary storage to the primary storage. External files, i.e. files not associated with the collection, will also be internalized (external files are those which you linked with Links : External file on the component menu). You can use this option, for example, if you want to speed up access to a small collection distributed on CD/DVD. Internalize files will move all files from CD/DVD to your hard disk. This way you will also be able to delete secondary storage path and give up using CD/DVD drive altogether
    • Repair - recover twin registries. A twin registry is a pair of registries in which each member of one registry points to one or more members of the other registry. For example, the phonetic transcription registry is a twin registry and is composed of two registries: words and word transcriptions
    • Delete unused - delete registry members that are not used anywhere in the collection (i.e. not linked to any component)
    • Size statistics - compute the size of texts, files, linked files, HTML objects, etc.
    • Add predefined templates - add predefined templates to the template registry (e.g. Item, Article, Multiple Choice, Spelling, etc.)
    • Add predefined styles - add all styles from the default stylesheets
    • Generate elements - insert all registry members into newly created elements (e.g. to convert the whole picture registry to a branch of pictures)
    • Generate references - generate references from member names that carry reference data

Selection

[edit | edit source]
  • Selection
    • Select all - select all registry members
    • Unselect all - unselect all registry members
    • Swap selection - unselect the selected registry members and select the unselected ones
    • Select index range - select members by their index range
    • Select position range - select members in a range of registry positions
    • Extract to subset (Shift+Ctrl+Enter) - display the currently selected registry members in a separate registry subset (selections are made by checking boxes to the left of registry member names)

Subset

[edit | edit source]
  • Subset
    • Previous subset (Alt+Left) - open the superset of the current registry subset
    • Next subset (Alt+Right) - open the nearest subset of the current registry subset. For example, if you do AND-search in the registry with Ctrl+S, you can browse through all search stages with Alt+Left and Alt+Right
    • Delete subset - delete the currently used registry subset. You cannot delete the first set which contains the entire registry
    • Match to subset - find all registry members whose users belong to a given element subset
    • Select format - create a registry subset with members of a specific file format
      • HTML - select subset of texts in the HTML format
      • Text (plain) - select subset of texts in the plain text format
      • EML - select subset of texts in the EML format
      • RTF - select subset of texts in the RTF format
      • BMP - select subset of images in the BMP format
      • JPEG - select subset of images in the JPEG format
      • GIF - select subset of images in the GIF format
      • PNG - select subset of images in the PNG format
    • Size range - select members falling into a selected size range (in bytes)
    • Select untranslated - generate a new registry subset in text registry with all members who have not yet been translated. This option is useful when you translate your collection and would like to quickly translate those texts that do not yet have translations associated in the translation registry

Use statistics

[edit | edit source]
  • Use statistics - generate a file with the number of users of individual members

Close

[edit | edit source]
  • Close (Ctrl+F4) - close the registry window

查看

[编辑|编辑源代码]
  • 查看
    • 列表元素- 列出所有元素使用当前选择的注册会员在元素列表中（右下角注册表窗口）
    • 查看元素- 显示元素在元素列表中选择或显示当前选择的第一个用户注册会员（如果未显示元素列表）。的元素显示在元素窗口不关闭注册表。请注意元素每次单击元素列表中的条目时都会显示（可以使用列表元素上面）
    • 转到元素- 与查看元素但是元素窗口被放在前面注册表窗口
    • 参考- 在图像中查看与当前显示的图像关联的参考注册表
    • 浏览当前（Ctrl+B) - 显示全部元素使用当前的注册会员在一个浏览器

    • 前往位置（Ctrl+G) - 前往注册会员通过提供其在注册表（例如文件 :修复合集通常会通过引用各个成员的实际位置来报告注册表问题）
    • 路径- 显示所选注册表对象文件（例如图像文件）的路径。显示文件的可用性小学和二级存储（如果这两者不同）、链接文件路径（如果有）和文件大小
    • 玩（Ctrl+F10) - 播放与当前相关的声音或视频注册会员以声音或视频的形式注册表
    • 编码- 显示注册表成员名称的UTF-8代码
    • 清除（ESC) - 此选项对于回溯您在注册表窗口。这是由以下人员完成的操作序列清除:
      1. 从列表中跳转元素到列表登记册成员（如果焦点位于元素列表上）
      2. 从对象编辑字段跳转到对象列表登记册成员（如果焦点位于编辑字段之一）
      3. 从成员列表跳转到顶部的编辑组合框注册表窗口（如果焦点在成员列表上）
      4. 清除顶部编辑组合框的内容注册表窗口如果焦点位于编辑组合框上
      5. 关闭注册表窗口如果焦点位于编辑组合框且组合框为空

编辑

[编辑|编辑源代码]

    • 粘贴- 将对象从剪贴板粘贴到注册表

    • 覆盖文件- 用另一个文件替换当前成员的目标文件。该选项类似于粘贴过来不同的是，虽然粘贴过来使用剪贴板中的数据，如果您使用覆盖文件，您指向磁盘上新版本文件的位置。

    • 编辑翻译（Ctrl+T) - 编辑与当前文本关联的翻译注册会员。请注意，在翻译注册表中，的含义编辑文字和编辑翻译是相反的，并且编辑文字实际上是用来编辑翻译的
    • 复制路径- 将路径复制到与当前关联的文件注册会员到剪贴板

搜索

[编辑|编辑源代码]
  • 搜索
    • 查找文本（Ctrl+S) - 查找当前的所有成员注册表子集其名称包含给定的字符串。将搜索结果放入新创建的注册表子集。使用Ctrl+S搜索全部登记册成员对于一个字符串。也可用于 AND 搜索
    • 先找到（Ctrl+F) - 找到第一个注册会员在名称包含给定字符串的当前子集中。搜索从当前所选成员之后的成员开始。你会发现快捷方式先找到,Ctrl+F，这里非常有用
    • 查找下一个（F3) - 重复最后一个先找到操作（您可以重复搜索F3）
    • 搜索和替换（Ctrl+R) - 查找当前的成员注册表子集其名称包含给定字符串并用另一个字符串替换该字符串。您可以对所有单个成员重复此操作并单独确认，或者一次性替换所有字符串。您可以进行全局搜索并替换您的文本收藏通过使用文本注册表。注意事项：所有注册表成员名称必须是唯一的，如果您的替换操作生成的名称已经存在，则 SuperMemo 将修改新名称以使其唯一。通常这会导致您的新名称末尾出现一些无意义的代码（例如地理#,地理@等）
    • 应用过滤器- 处理当前文本注册表与一个过滤文件（例如，将一组字符串替换为另一组字符串、删除分隔注释等）。

工具

[编辑|编辑源代码]
  • 工具
    • 导入文件- 将多个文件导入到注册表。例如，如果您有一组现成的图片或声音文件，您可以将它们分别导入到图像或声音注册表中，然后再在您的应用程序中使用它们。元素
    • 导入文本- 导入带有注册表成员名称的文本导出为文本（下）。例如，拼写检查后
    • 导出文件- 导出与当前成员关联的文件
    • 导出为文本- 将所有注册表成员名称导出到文本文件中。大多数情况下，您会想要对文件的内容进行拼写检查。注册表用你最喜欢的拼写检查器
    • 按频率排序- 排序注册表对于数量元素使用个人会员
    • 按尺寸排序- 排序注册表对于与单个成员相关的文件的大小（例如，从臃肿的文件中删除最大的图片）收藏）
    • 按优先级排序- 排序注册表为会员优先。这里的优先级由比率决定：（用户数）/（文件大小）。如果您想要查找占用最多空间且最少使用的成员（例如，高级英语97声音文件已使用此选项进行修剪，以适应 650MB CD-ROM 大小限制）
    • 外部化文件- 将所有注册表文件导出到二级存储
    • 内化文件- 移动与该相关的所有文件注册表从二级存储到主存储。外部文件，即与外部文件无关的文件收藏，也将被内部化（外部文件是您链接的文件友情链接 ：外部文件于组件菜单）。例如，如果您想加快对小型文件的访问速度，则可以使用此选项收藏以 CD/DVD 形式发行。内化文件会将所有文件从 CD/DVD 移动到硬盘。这样您还能够删除辅助存储路径并完全放弃使用 CD/DVD 驱动器
    • 维修- 恢复双胞胎注册表。孪生注册表是一对注册表，其中一个注册表的每个成员都指向另一个注册表的一个或多个成员。例如，音标注册表是一个孪生注册表，由两个注册表组成：单词和单词转录
    • 删除未使用的- 删除登记册成员没有在任何地方使用收藏（即不链接到任何组件）
    • 尺寸统计- 计算文本、文件、链接文件、HTML 对象等的大小。
    • 添加预定义模板- 添加预定义的模板到模板注册表（例如项目,文章,多项选择,拼写等）
    • 添加预定义样式- 添加默认样式表中的所有样式
    • 生成元素- 插入全部登记册成员进入新创建的元素（例如将整个图片注册表转换为图片分支）
    • 生成参考文献- 从携带参考数据的成员名称生成参考
