With the universal adoption of Unicode (since SuperMemo 15), fonts should no longer be a headache as was the case in earlier versions of SuperMemo.

There is still a confusing array of choices of components and encodings. If in doubt, use the default HTML component. To start using fonts and richly formatted texts, you need to change nothing.

Why do I see so many different font sizes in SuperMemo?

[edit | edit source]

If you have ever wondered why SuperMemo displays texts in so many seemingly inconsistent styles and sizes, the answer lies in the fact that SuperMemo uses several distinct fonts for different purposes:

  • interface font — the main font you interact with (most likely the one that bothers you)
  • menu font — handled by the VCL framework via Windows, not directly by SuperMemo
  • caption font — handled by Windows, not SuperMemo
  • contents font — used for displaying titles in the Contents window, Ancestors, Registry, etc. (useful e.g. for multilingual collections)
  • collection font — the font used inside components, which can span all imaginable typefaces and sizes

To further complicate matters, SuperMemo sometimes needs to squeeze text into small areas (e.g. narrow panels or compact layouts). In such cases, the interface font may be automatically reduced in size. This is the source of the apparent mosaic of font sizes you may notice across different windows. It is by design, not a bug.

If you still want to explore all options available, you may wish to know that the way you use fonts in SuperMemo depends on the type of the component, the way the font is defined, and the way the texts are encoded. The number of possible combinations of these three may lead to substantial confusion (not only among SuperMemo beginners). If fonts and encodings are important for your learning material (e.g. if you learn foreign languages using non-Latin alphabets), it is important that you make good choices in the way you use fonts before you embark on a major expansion of your collection. The summary overview below will help you understand the terminology of fonts as well as the pros and cons of different choices.

Where are the fonts defined?

[edit | edit source]

The way you see fonts in SuperMemo can be defined in a number of ways. Here are the most often used methods:

  1. HTML (WYSIWYG): Inside the HTML component or WebView component, which is the default text component, you can change fonts by choosing options from the Text : Font submenu of the component menu, or using shortcuts such as Ctrl+B for bold, Ctrl+] to increase the size of the font, etc. Note that those operations may override your stylesheet settings and might prevent global changes to fonts.
  2. HTML code: If you understand HTML, you can edit texts using a component's HTML source code (e.g. by using Ctrl+Shift+F6 over the component). Note that this method may override your stylesheet settings and might prevent global changes to fonts.
  3. Font registry: The font registry can be used to store reusable fonts (e.g. for plain text components, templates, or frequently used font definitions for HTML). The font registry can be inspected with Search : Other registries : Font. You can use Text : Font : Edit font in text components to modify a font linked from the registry. Text : Font : New font can be used to define a new font. Finally, you can use Text : Font : Link font to link a component with an existing font definition in the registry. The registry and the font definitions are stored in the [REGISTRY]/font.* files. A definition of a registry-stored font in SuperMemo includes the typeface (e.g. Arial, Verdana, etc.), size, color, and style (e.g. bold, italic, etc.).
  4. Options: Toolkit : Options : Fonts makes it possible to define default fonts, the font displayed in the Contents window, interface font, etc. If you check Use question and answer fonts in HTML, you can define the font originally used when starting a new question or a new answer. Note that this font will override the stylesheet used in HTML components. Definitions of fonts defined in Options are stored in INI files: [BIN]\supermemo.ini (if SuperMemo specific) and collection.ini (if collection specific).
  5. MSHTML (HTML component): The default font for HTML components rendered via the MSHTML engine can be influenced through Windows settings. This font will affect HTML texts that do not define the <font> tag (i.e. have not been previously formatted) and do not use any stylesheet. Note that Accessibility settings in Windows can also affect the way fonts are displayed in HTML components.
  6. Stylesheets: Stylesheets in SuperMemo can be used to define fonts in HTML components and WebView components. Several classes can have their font defined (e.g. texts, extracted texts, search highlights, references, etc.) in a SuperMemo stylesheet. The default stylesheet is defined in Toolkit : Options : Fonts : Stylesheet. Individual components or templates can have their stylesheet changed with options on the component menu listed at Text : Style submenu. Stylesheets are your best way to globally change fonts in HTML components and WebView components.

How are texts encoded?

[edit | edit source]

Character encoding is a code that matches a set of characters with a specific sequence of numbers. Computers store all texts as numbers. For example, in ASCII encoding, numbers 97, 98, and 99 code for characters a, b, and c. Thus a sequence 97-98-99 codes for abc. SuperMemo uses 3 basic character encoding systems: ANSI, UTF-8 and 16-bit Unicode. ANSI is used to represent simple texts made of up to 255 characters taken from any number of 255-character sets. In ANSI encoding, interpretation of individual codes depends on the character encoding table called a code page. Unicode is used to represent texts with virtually all commonly used characters. UTF-8 encoding is used to represent Unicode characters with 8-bit characters in such a way that all ASCII texts are unchanged and all non-ASCII characters have the most significant bit set.

Character encodings used in SuperMemo:

  • ANSI: ANSI is only used in SuperMemo for compatibility with older versions of SuperMemo. Those texts cannot use more than 255 characters, but use minimum space. Fonts used with this encoding are also referred to as OEM fonts and can be defined with the help of code pages (some older collections may still be based on code pages and OEM fonts). Internally, SuperMemo converts all character strings to Unicode (UTF-16).
  • Unicode: UTF-16 encoding is used throughout SuperMemo as the default encoding. UTF-16 texts can use virtually all commonly used characters. HTML components represent Unicode characters using HTML codes. For example: the letter µ is represented as a sequence of ASCII characters: &#181; where 181 is its Unicode number. You can inspect the HTML representation of non-ASCII characters in HTML components with Shift+Ctrl+F6 (File : View source).
  • UTF-8: To reduce the size of collections, UTF-8 is used to encode Unicode characters in the text registry (Search : Texts). This makes it possible to uniformly encode all ANSI and Unicode texts and quickly search them using the same fast low-level search procedure (e.g. using Search : Find elements). Use View : Encoding on the registry menu to inspect all encodings of a given registry member text:
    • Unicode name (i.e. the way the text is displayed for you),
    • Unicode double-byte codes (i.e. the numeric way the text is encoded internally in SuperMemo), and
    • UTF-8 encodings (i.e. the way the text is stored on the disk).

Some Q&A files may also be UTF-8 encoded. If so, you should decode these files on Q&A import (check Decode UTF-8 box in the Import Text dialog box).

为什么我在 SuperMemo 中看到这么多不同的字体大小？

[编辑|编辑源代码]

如果您曾经想知道为什么 SuperMemo 以如此多看似不一致的样式和大小显示文本，答案就在于 SuperMemo 使用几种不同的字体来实现不同的目的：

  • 界面字体- 您与之交互的主要字体（很可能是困扰您的字体）
  • 菜单字体— 由 VCL 框架通过 Windows 处理，而不是直接由 SuperMemo 处理
  • 标题字体— 由 Windows 处理，而不是 SuperMemo
  • 内容字体— 用于显示标题内容窗口、祖先、注册表等（例如对于多语言集合有用）
  • 收藏字体— 里面使用的字体组件，可以涵盖所有可以想象的字体和尺寸

更复杂的是，SuperMemo 有时需要将文本压缩到较小的区域（例如狭窄的面板或紧凑的布局）。在这种情况下，界面字体可能会自动缩小。这是您可能会在不同窗口中注意到的明显的字体大小马赛克的来源。这是设计使然，不是错误。

如果您仍然想探索所有可用选项，您可能希望知道在 SuperMemo 中使用字体的方式取决于字体的类型组件、字体的定义方式以及文本的编码方式。这三者可能的组合数量可能会导致严重的混乱（不仅在 SuperMemo 初学者中）。如果字体和编码对您的学习材料很重要（例如，如果您使用非拉丁字母学习外语），那么在开始大规模扩展您的学习材料之前，在使用字体的方式上做出良好的选择就很重要。收藏。下面的摘要概述将帮助您了解字体的术语以及不同选择的优缺点。

文本是如何编码的？

[编辑|编辑源代码]

字符编码是将一组字符与特定数字序列相匹配的代码。计算机将所有文本存储为数字。例如，在ASCII码编码，数字 97、98 和 99 字符代码一个,乙, 和c。因此序列 97-98-99 编码为ABC。 SuperMemo 使用 3 种基本字符编码系统：美国国家标准协会,UTF-8和 16 位统一码。 ANSI 用于表示由任意数量的 255 个字符集中的最多 255 个字符组成的简单文本。在 ANSI 编码中，各个代码的解释取决于称为字符编码表代码页。统一码用于表示几乎所有常用字符的文本。UTF-8编码用于用 8 位字符表示 Unicode 字符，所有 ASCII 文本均保持不变，所有非 ASCII 字符均具有最高有效位集。

SuperMemo 中使用的字符编码：

  • 美国国家标准协会：ANSI 仅在 SuperMemo 中使用，以兼容旧版本的 SuperMemo。这些文本不能使用超过 255 个字符，但使用最小空间。与此编码一起使用的字体也称为OEM字体并且可以在代码页的帮助下定义（一些较旧的收藏品可能仍基于代码页和 OEM 字体）。 SuperMemo 在内部将所有字符串转换为 Unicode (UTF-16)。
  • 统一码：整个 SuperMemo 都使用 UTF-16 编码作为默认编码。 UTF-16 文本几乎可以使用所有常用字符。HTML 组件使用 HTML 代码表示 Unicode 字符。例如：字母 µ 表示为 ASCII 字符序列：&#181;其中 181 是它的 Unicode 编号。您可以检查非 ASCII 字符的 HTML 表示形式HTML 组件与Shift+Ctrl+F6（文件 ：查看源码）。
  • UTF-8: 缩小尺寸收藏品，UTF-8用于对文本中的Unicode字符进行编码注册表（搜索 :文本）。这使得统一编码所有 ANSI 和 Unicode 文本并使用相同的快速低级搜索过程（例如使用搜索 :查找元素）。使用查看 ：编码于注册表菜单检查给定注册表成员文本的所有编码：
    • Unicode 名称（即文本显示方式），
    • Unicode 双字节代码（即 SuperMemo 中文本内部编码的数字方式），以及
    • UTF-8 编码（即文本在磁盘上存储的方式）。

某些问答文件也可能是 UTF-8 编码的。如果是这样，您应该在问答导入时解码这些文件（检查解码UTF-8框中的导入文本对话框）。
